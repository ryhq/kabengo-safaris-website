# 07 — Next.js Frontend Deployment (cPanel)

Deploy the Next.js frontend to cPanel shared hosting using the Node.js Selector (Phusion Passenger).

---

## Prerequisites

- cPanel hosting with **Node.js Selector** (Passenger) enabled
- Node.js 18+ available on the server
- A subdomain configured in cPanel (e.g., `next.kabengosafaris.com`)
- The backend API is live at `https://api.kabengosafaris.ryhqtech.com/api`

---

## 1. Configure Next.js for Standalone Output

In `next.config.ts`:

```typescript
import createNextIntlPlugin from "next-intl/plugin";

const withNextIntl = createNextIntlPlugin();

const nextConfig: NextConfig = {
  output: "standalone",
  images: {
    remotePatterns: [
      {
        protocol: "https",
        hostname: "*.kabengosafaris.com",
        pathname: "/**",
      },
    ],
  },
};

export default withNextIntl(nextConfig);
```

The `output: "standalone"` setting produces a minimal, self-contained build.

---

## 2. Set the Production API URL

In `.env.local`:

```env
NEXT_PUBLIC_API_BASE_URL=https://api.kabengosafaris.ryhqtech.com/api
```

> `NEXT_PUBLIC_*` variables are baked into the build at build time. Set this **before** building.

---

## 3. Build the Project

```bash
npm run build
```

The standalone output is at `.next/standalone/` and contains:
- `server.js` — Node.js entry point
- `node_modules/` — minimal dependencies
- `.next/` — compiled app (partial)
- `package.json`

---

## 4. Prepare the Deployment Package

The standalone output is missing static assets and the `public/` folder. Copy them in:

```bash
# Copy static assets
cp -r .next/static .next/standalone/.next/static

# Copy public folder
cp -r public .next/standalone/public

# Create zip for upload
cd .next/standalone
zip -r deploy.zip .
```

---

## 5. Create the Subdomain in cPanel

1. Go to **cPanel → Domains**
2. Click **Create A New Domain**
3. Enter your subdomain: `next.kabengosafaris.com`
4. Document root auto-fills (e.g., `/home/username/next.kabengosafaris.com`)
5. Click **Submit**

---

## 6. Upload and Extract

1. Go to **cPanel → File Manager**
2. Navigate to the subdomain's document root
3. Click **Upload** → upload `deploy.zip`
4. Select `deploy.zip` → click **Extract** → extract to current directory
5. Delete `deploy.zip` after extraction

Final directory structure:

```
next.kabengosafaris.com/
├── .next/
│   ├── server/
│   └── static/
├── node_modules/
├── public/
├── package.json
└── server.js
```

---

## 7. Configure the Node.js Application

1. Go to **cPanel → Setup Node.js App**
2. Click **CREATE APPLICATION**
3. Fill in:

| Field | Value |
|-------|-------|
| Node.js version | 18.x or 20.x |
| Application mode | **Production** |
| Application root | `next.kabengosafaris.com` |
| Application URL | Select `next.kabengosafaris.com` from dropdown, leave text field empty |
| Application startup file | `server.js` |

4. (Optional) Add environment variables for any runtime (non-`NEXT_PUBLIC_`) vars
5. Click **CREATE**

---

## 8. Start the App

- Click **RESTART** on the Node.js panel
- Click **OPEN** to test the URL

---

## 9. Enable HTTPS

1. Go to **cPanel → SSL/TLS** or **AutoSSL**
2. Ensure SSL certificate is issued for your subdomain
3. In **cPanel → Domains**, toggle **Force HTTPS Redirect** to **On**

---

## Redeployment

When you make changes and need to redeploy:

```bash
# 1. Build locally
npm run build

# 2. Prepare standalone package
cp -r .next/static .next/standalone/.next/static
cp -r public .next/standalone/public

# 3. Zip
cd .next/standalone
zip -r deploy.zip .

# 4. Upload to cPanel File Manager (overwrite existing files)
# 5. Extract (check "Overwrite existing files")
# 6. Restart the Node.js app in cPanel
```

## Troubleshooting

### "Index of /" directory listing

The Node.js app isn't routing through Passenger:
- Check **Application URL** matches your subdomain exactly
- Verify `.htaccess` has Passenger directives (auto-generated by cPanel)
- Click **RESTART** after changes

### CORS errors

The backend must allow the frontend's origin. In Spring Boot `SecurityConfigurations.java`:

```java
corsConfiguration.setAllowedOriginPatterns(List.of("*"));
```

### Blank page or crash

1. Check Passenger logs (path shown in Node.js panel)
2. SSH in and test manually:
   ```bash
   source /home/username/nodevenv/next.kabengosafaris.com/18/bin/activate
   cd /home/username/next.kabengosafaris.com
   node server.js
   ```
3. Ensure `.next/static/` exists inside the app root

### Environment variables not working

- `NEXT_PUBLIC_*` vars are embedded at **build time** — rebuild if you change them
- Non-prefixed env vars are read at **runtime** from the Node.js panel
