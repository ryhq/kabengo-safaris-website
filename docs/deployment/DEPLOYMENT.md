# Deploying Next.js to cPanel (Node.js + Standalone)

This guide covers deploying a Next.js app (with middleware, e.g. `next-intl`) to a shared hosting cPanel environment using the Node.js Selector (Phusion Passenger).

---

## Prerequisites

- cPanel hosting with **Node.js Selector** (Passenger) enabled
- Node.js 18+ available on the server
- A subdomain or domain configured in cPanel
- SSH or Terminal access (optional but helpful)

---

## 1. Configure Next.js for Standalone Output

In `next.config.ts`, add the `output: "standalone"` option:

```ts
const nextConfig: NextConfig = {
  output: "standalone",
  // ... other config
};
```

This tells Next.js to produce a minimal, self-contained build in `.next/standalone/` that includes only the necessary `node_modules`.

---

## 2. Set Environment Variables

Create or update `.env.local` with your production API URL:

```env
NEXT_PUBLIC_API_BASE_URL=https://your-api-domain.com/api
```

> **Important:** `NEXT_PUBLIC_*` variables are baked into the build at build time, so set them **before** building.

---

## 3. Build the Project

```bash
npm run build
```

After building, the standalone output is at `.next/standalone/`. It contains:
- `server.js` — the Node.js entry point
- `node_modules/` — minimal dependencies
- `.next/` — compiled app (partial, missing static assets)
- `package.json`

---

## 4. Prepare the Deployment Package

The standalone output does **not** include static assets or the `public/` folder. You must copy them manually:

```bash
# Copy static assets into standalone
cp -r .next/static .next/standalone/.next/static

# Copy public folder into standalone
cp -r public .next/standalone/public
```

Then create a zip for upload:

```bash
cd .next/standalone
zip -r deploy.zip .
```

---

## 5. Create the Subdomain in cPanel

1. Go to **cPanel → Domains**
2. Click **Create A New Domain**
3. Enter your subdomain (e.g., `next.yourdomain.com`)
4. Document root will auto-fill (e.g., `/home/username/next.yourdomain.com`)
5. Click **Submit**

---

## 6. Upload and Extract Files

1. Go to **cPanel → File Manager**
2. Navigate to your subdomain's document root (e.g., `/home/username/next.yourdomain.com/`)
3. Click **Upload** → upload `deploy.zip`
4. Select `deploy.zip` → click **Extract** → extract to the current directory
5. Delete `deploy.zip` after extraction

Your directory should now contain:
```
next.yourdomain.com/
├── .next/
│   ├── server/
│   └── static/
├── node_modules/
├── public/
├── package.json
└── server.js
```

---

## 7. Create the Node.js Application

1. Go to **cPanel → Setup Node.js App**
2. Click **CREATE APPLICATION**
3. Fill in:

| Field                    | Value                          |
|--------------------------|--------------------------------|
| Node.js version          | 18.x or 20.x                  |
| Application mode         | **Production**                 |
| Application root         | `next.yourdomain.com`          |
| Application URL          | Select `next.yourdomain.com` from dropdown, leave text field empty |
| Application startup file | `server.js`                    |
| Passenger log file       | Leave default                  |

4. Add environment variables (click **ADD VARIABLE**):

| Name                        | Value                              |
|-----------------------------|------------------------------------|
| `NEXT_PUBLIC_API_BASE_URL`  | `https://your-api-domain.com/api`  |

> **Note:** Since `NEXT_PUBLIC_*` is baked at build time, the env var here is mainly for reference. If your app uses server-side runtime env vars (without `NEXT_PUBLIC_` prefix), they **will** be read from here.

5. Click **CREATE**

---

## 8. Start / Restart the App

- Click **RESTART** on the Node.js panel
- Click **OPEN** to test the URL
- Optionally enable **Force HTTPS Redirect** in **cPanel → Domains** for your subdomain

---

## 9. Enable HTTPS (SSL)

1. Go to **cPanel → SSL/TLS** or **AutoSSL**
2. Ensure your subdomain has an SSL certificate
3. In **cPanel → Domains**, toggle **Force HTTPS Redirect** to **On**

---

## Troubleshooting

### "Index of /" directory listing instead of app

The Node.js app isn't routing traffic through Passenger. Check:
- **Application URL** dropdown matches your subdomain exactly
- The `.htaccess` in your subdomain root has Passenger directives (auto-generated by cPanel)
- Click **RESTART** after any URL changes

### CORS errors from API

Your backend must allow your frontend's origin. In Spring Boot:

```java
corsConfiguration.setAllowedOriginPatterns(List.of("*"));
```

Or explicitly list origins:

```java
corsConfiguration.setAllowedOrigins(List.of(
    "https://next.yourdomain.com",
    "https://yourdomain.com",
    "http://localhost:3000"
));
```

### App crashes or blank page

1. Check the Passenger log file (path shown in Node.js panel)
2. Via SSH, enter the virtual environment and test manually:
   ```bash
   source /home/username/nodevenv/next.yourdomain.com/18/bin/activate
   cd /home/username/next.yourdomain.com
   node server.js
   ```
3. Ensure `.next/static/` exists inside the app root

### Environment variables not working

- `NEXT_PUBLIC_*` vars are embedded at **build time**, not runtime. Rebuild and redeploy if you change them.
- Non-prefixed env vars (e.g., `DATABASE_URL`) are read at runtime from the Node.js panel's environment variables.

---

## Redeployment

When you make changes and need to redeploy:

```bash
# 1. Build locally
npm run build

# 2. Prepare standalone package
cp -r .next/static .next/standalone/.next/static
cp -r public .next/standalone/public

# 3. Zip
cd .next/standalone
zip -r deploy.zip .

# 4. Upload to cPanel File Manager (overwrite existing files)
# 5. Extract (check "Overwrite existing files")
# 6. Restart the Node.js app in cPanel
```
